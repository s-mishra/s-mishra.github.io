# s-mishra.github.io

Personal academic website for **Swapnil Mishra**.
Live at **https://s-mishra.github.io** and **https://www.smishra.dev**.

This is a small, dependency-free **static site generated by `build.mjs` (Node)**.
It replaced the old Hugo/Academic build, which stopped working with current Hugo
versions (the cause of the broken auto-deploy). The old Hugo source still lives in
this repo (`config/`, `content/`, `themes/`, `static/`) for reference but is no
longer used.

## How deployment works (two branches)

| Branch      | Role                                                                   |
|-------------|------------------------------------------------------------------------|
| `main`      | **Source** — you edit here. Push to it to deploy.                       |
| `gh-pages`  | **Built output** — generated automatically. Don't edit by hand.        |

On every push to `main`, the GitHub Action `.github/workflows/gh-pages.yml`
runs `node build.mjs` and publishes `public/` to `gh-pages`, which GitHub Pages
(and Netlify, for the custom domain) serve. **So updating the site = edit data,
commit, push to `main`.**

## Update the content

All content lives in `data/*.json` — no HTML editing needed.

- **`data/profile.json`** — name, titles, bio, research interests, education,
  appointments, honours/awards, and social links.
- **`data/publications.json`** — the publication list (see below).
- **`assets/img/profile.jpg`** — profile photo (replace this file to change it).
- **`assets/files/cv.pdf`** — CV (replace this file to update).

### Add or feature a publication

Easiest: open `data/publications.json` and add an object (newest first):

```json
{
  "title": "Paper title",
  "authors": [{"given": "S", "family": "Mishra"}, {"given": "A", "family": "Coauthor"}],
  "venue": "Nature Communications",
  "year": 2026,
  "doi": "10.1038/xxxxx",
  "kind": "journal",
  "featured": true
}
```

`kind` is one of `journal | conference | chapter | thesis`; `featured: true`
shows the paper on the homepage. Your own name
(`{"family": "Mishra", "given": "S"}`) is automatically **bold-ed**.

**Or regenerate from a fresh export** (recommended for bulk updates): export
BibTeX from your [NUS Discovery profile](https://discovery.nus.edu.sg/24081-swapnil-mishra/publications)
or Google Scholar, drop the `.bib` file(s) into `data/sources/`, then:

```bash
python3 scripts/build_publications.py     # regenerates data/publications.json
```

Featured papers are kept via `FEATURED_DOIS` at the top of that script.

## Build & preview locally

```bash
node build.mjs            # writes ./public
open public/index.html    # preview in a browser
```

## Deploy

```bash
git add -A && git commit -m "Update content" && git push origin main
```

The Action deploys within ~1 minute. To deploy manually instead, run
`./deploy.sh`.
